The Anatomy of Neutralization

How It Works

Toxic Shield operates with sub-millisecond precision, leveraging a fine-tuned ML pipeline to classify content before it ever reaches the user's display.

System Pipeline

From Comment to Decision

1

User Comment

A comment is submitted — from the website demo, API, or Chrome extension.

2

Pre-processing Layer

Automated text normalization: lowercase, leet-speak decoding (0→o, 1→i, @→a), URL removal, username stripping, emoji filtering.

3

FastAPI Backend

The comment is sent via POST to /predict_batch. Hard-rule pre-filter checks for extreme phrases first.

4

Feature Extraction

Word TF-IDF (20k features) + Character TF-IDF (20k features) + Profanity features (2). Total: 40,002 dimensions.

5

ML Classification

Logistic Regression with calibrated probabilities returns a continuous toxicity score from 0.0 to 1.0.

6

Toxicity Decision

Score ≥ threshold → Toxic (comment hidden/flagged). Score < threshold → Safe (comment displayed normally).

API Response

Prediction Payload

// POST /predict_batch
{
  "texts": ["go kill yourself", "great video!"]
}

// Response

  "threshold": 0.5

// score < 0.5 → SAFE → ALLOW

Core Components

Built for Real-time Digital Moderation

📊

Dataset Training

Architected on 180,000+ hand-labeled samples from Jigsaw & HateXplain. 420 synthetic soft-insult examples oversampled 10× for indirect toxicity detection.

🔬

Feature Extraction

TF-IDF vectorization transforms raw characters into a 40,002-dimensional mathematical space, catching disguised insults that keyword filters miss entirely.

🤖

ML Classification

Logistic Regression with calibrated probabilities. Sub-1ms inference latency, making moderation invisible to the end-user while providing maximum security.

⚡

Hard Rule Pre-filter

O(k) exact match against a lexicon of extreme phrases fires before the model, guaranteeing zero-latency blocking of the most severe content.

🔤

Leet-speak Normalization

Character substitution map (0→o, 1→i, @→a, $→s, !→i) deobfuscates toxic text before vectorization, defeating evasion attempts.

🌐

Edge Deployment

Lightweight enough to run in local browser extensions or low-power serverless functions without performance degradation.
