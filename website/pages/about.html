About the Project

Why we built Toxic Comment Shield, what problems it solves, and the full story of its development across 10 phases.

The Problem

Online Toxicity is a Growing Crisis

Online comment sections on platforms like YouTube, Reddit, and Instagram are frequently polluted with toxic language — insults, hate speech, and harassment — that degrades the experience for users. Existing platform-level moderation is reactive, inconsistent, and often too slow.

Research shows that exposure to toxic comments reduces user engagement, drives away marginalized communities, and contributes to a measurable decline in mental health for those targeted.

Platform-level moderation systems are opaque, centralized, and can take hours or days to act — far too slow for real-time conversations.

The Solution

User-controlled AI Moderation

Toxic Comment Shield gives users control by running toxicity detection locally in their browser. Comments are sent in batches to a lightweight FastAPI backend, scored by a trained ML model, and hidden automatically — with a click-to-reveal option.

⚡

Speed

Under 1ms per comment using classical ML — no transformers needed.

🎯

Accuracy

Trained on 180,000+ real-world and synthetic samples from multiple datasets.

🔒

Privacy

No data is stored — comments are scored and discarded immediately. Everything runs locally.

Development History

Built Across 10 Phases

Phase 1

Rule-Based Filtering

Keyword-based toxic word list. Extension skeleton: manifest, content.js, popup.

Phase 2

ML Backend Integration

First FastAPI backend with TF-IDF + Logistic Regression. Real ML scores replace keyword matching.

Phase 3

Real-time Detection

Replaced polling with MutationObserver. Zero-delay detection of dynamically loaded comments.

Phase 4

Extension UI

Dark popup with toggle + sensitivity slider. State persisted via chrome.storage.sync.

Phase 5

Probability Scores

Moved from predict() to predict_proba(). Tunable threshold — users choose their tolerance.

Phase 6

Multi-Platform Support

Site adapter system for YouTube, Reddit, Instagram. One codebase, three platforms.

Phase 7

Performance Optimization

Client-side cache + batch queue. 200 comments → 4 API calls instead of 200.

Phase 8

Advanced Feature Engineering

Combined Word + Char TF-IDF + Profanity. 40,002-dimensional feature space.

Phase 9

Dataset Expansion

Merged Jigsaw + HateXplain + 4,200 synthetic soft insults. 180,000+ samples total.

Phase 10

Leet-speak Normalization

LEET_MAP character substitution (0→o, 1→i, @→a). Detection improved from 6/10 to 8/10.
